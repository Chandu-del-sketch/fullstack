# Git Workflow

**Check Repository status** 

`git status`

- here you will see whether there is an repository or you have to create new one.

**Add changes to staging**

`git add .`

- here you can add the changes if you can do any changes in code or anything.

**Commit changes with message**

`git commit -m "Commit message"`

- by using this command you can commit the changes and it will directly affected to the code which is already pushed.

**Push Changes Github**

`git origin main`(if you have main branch) or `git origin master`(is you have a master branch)

- it push the entire project into the git hub.



# History and Diff

**Git Log**
- Shows the history of commits.
- Tells who changed what and when.
- Used to track project progress.

Command - `git log` 

**Git Diff**
- Shows differences in code
- Compares:
    modified files <-> last commit
- Used before committing to review changes.

Command - `git diff`

**Git Show**
- Shows details of a single commit.
- Displays:
    commit message
    author
    exact code changes

Command - `git show`
